# 知识管理与引用

科研智能体一旦真的开始辅助工作，最先需要“资产化”的往往不是模型，而是你的文献库、批注、Bib、实验记录和项目笔记。

## 1. 个人文献库与引用管理

| 资源 | 类型 | 简述 | 链接 |
| --- | --- | --- | --- |
| Zotero | 文献管理器 | 最值得作为个人科研资产底座的工具之一。 | [Zotero](https://github.com/zotero/zotero) |
| Better BibTeX for Zotero | Zotero 插件 | BibTeX 导出、引用键管理、自动更新等方面非常实用。 | [retorquere/zotero-better-bibtex](https://github.com/retorquere/zotero-better-bibtex) |
| zotero-mcp | MCP 接入层 | 让 Agent 能查询 Zotero 条目、笔记和 BibTeX。 | [54yyyu/zotero-mcp](https://github.com/54yyyu/zotero-mcp) |
| Papis | CLI 文献管理 | 如果你偏终端工作流，Papis 是个值得关注的轻量替代路线。 | [papis/papis](https://github.com/papis/papis) |

## 3. 我更推荐的资产组织方式

### 层 1：原始资产

- 论文 PDF
- Bib 条目
- 原始网页与 DOI
- OpenReview / arXiv / OpenAlex 链接

### 层 2：结构化资产

- paper notes
- comparison table
- claim list
- baseline list
- novelty report

### 层 3：项目级资产

- hypotheses
- experiment plans
- run logs
- draft / response letter

如果未来要让 Agent 长期服务科研，关键不是“能不能回答问题”，而是“能不能把这些层级稳定保存下来并可追溯”。

## 4. 对 ResearchOS 的建议

更稳的知识层组合通常是：

1. `Zotero` 作为个人文献资产层
2. `Better BibTeX` 负责引用与导出稳定性
3. `zotero-mcp` 负责把个人库接给 Agent
4. `Notion / GitHub` 作为项目级知识与协作层

这会比只依赖聊天上下文可靠得多。
