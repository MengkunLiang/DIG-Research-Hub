# 文献检索与单篇精读

这一页适合在“已经有主题词或 seed paper，需要迅速建立稳定论文池和精读卡片”时使用。

## 一、什么时候进入这一页

- 已经有方向，但论文池还比较散。
- 已经有一批检索结果，但不知道哪些值得深读。
- 需要把 PDF、附录、公式和表格转成结构化材料。

## 二、先按这个顺序看

1. [02-文献检索与单篇精读](../02-文献检索与单篇精读/README.md)
2. [02-文献检索与单篇精读/精读用Skills、MCP与解析工具详表](../02-文献检索与单篇精读/精读用Skills、MCP与解析工具详表.md)
3. [95-文献检索与知识管理/文献检索与阅读](../95-文献检索与知识管理/文献检索与阅读.md)
4. [95-文献检索与知识管理/PDF解析与阅读加工](../95-文献检索与知识管理/PDF解析与阅读加工.md)
5. [知识管理与证据沉淀](./知识管理与证据沉淀.md)

## 三、最小使用组合

| 资源类型 | 推荐先用 |
| --- | --- |
| Skills | `paper-glance-skill`、`research-lookup`、`academic-paper-reviewer` |
| MCP | `Paper Search MCP`、`OpenAlex MCP`、`OpenReview MCP`、`Zotero MCP`、`arXiv LaTeX MCP` |
| 解析工具 | `GROBID`、`Marker`、`Docling`、`MinerU`、`Nougat` |
| Agent / 工作台 | `PaperQA2`、`RTI`、`Open Research`、`STORM` |

## 四、推荐的阅读路径

1. 用 `Paper Search MCP + OpenAlex / Semantic Scholar` 建立候选论文池。
2. 用 `paper-glance-skill` 做第一轮统一结构化拆解。
3. 对核心论文再用 `GROBID / Marker / Docling / MinerU` 做深层解析。
4. 把值得保留的结论、页码、图表与局限转入 [知识管理与证据沉淀](./知识管理与证据沉淀.md)。

## 五、完成这一步的最低产出

1. 一个稳定的核心论文池。
2. 每篇论文至少一张结构化精读卡。
3. 明确的可比较维度：问题、方法、数据、结果、局限。
4. 一批可回查的证据条目，而不只是摘要。

## 六、下一步往哪里走

- 如果材料已经很多，先进入 [知识管理与证据沉淀](./知识管理与证据沉淀.md)。
- 如果要判断拥挤度、差异点和 reviewer concern，进入 [新颖性判断与综述](./新颖性判断与综述.md)。
