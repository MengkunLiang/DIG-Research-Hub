# 收录与去重原则

## 收录原则

- 优先官方文档、官方仓库、论文主页、长期维护项目与高信号公开综述。
- 优先直接服务科研工作流的资源，而不是泛工具清单。
- 中文说明优先，收录对象以全球公开资源为主。
- 同时关注 `Skills / MCP / Agent / Prompt / 数据源 / Benchmark / 文稿工程` 等不同层次。
- 尽量优先主归属页写完整说明，其他页面只保留短说明与内链。

## 统一收录字段

建议尽量按下面几个字段写新条目：

1. `类型`
2. `适合环节`
3. `一句话说明`
4. `链接`
5. `主归属页`

## 去重原则

- 同一外部资源只保留一个主归属页做较完整说明。
- 其他页面优先用一句话摘要加内链，不重复堆长表。
- 顶层任务页负责组合展示，附录页负责类型查全。
- `98-全局索引` 负责快速检索，不重复承担正文说明。

## 分层原则

- `01-06` 主模块：回答“当前这一步该怎么做”，优先保留最小工具栈和高频资源。
- `07-09` 横向层：负责治理、方向补充和任务化使用面板。
- `91-97` 类型附录：负责系统查全，不重复承担任务化叙事。
- `98` 全局索引：负责横向快查、外部导航和入口汇总。

## 外部资料整合方式

- 外部公开仓库优先采用中文摘要、任务化拆分和原链接回指。
- 对超长 prompt 库、教程页和资源合集，优先抽取有效分类和使用入口，不做整页镜像。
- 失效链接可作为历史参考保留，但应标注当前状态与核验日期。

## 维护建议

- 新资源先判断主归属页，再决定是否同步到索引层。
- 大型合集优先拆成“高价值入口 + 主归属页回链”，不直接复制成长清单。
- 每轮补库后更新 [更新日志](./更新日志.md) 和 [补库规则与待补清单](./补库规则与待补清单.md)。
