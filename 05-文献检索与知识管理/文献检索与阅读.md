# 文献检索与阅读

这一页聚焦“怎么把公开学术世界接给 Agent”。

## 1. 学术平台与开放 API

| 资源 | 类型 | 简述 | 链接 |
| --- | --- | --- | --- |
| OpenAlex | 开放学术图谱 | 很适合做 topic landscape、引文网络、作者和机构分析。 | [OpenAlex](https://openalex.org/) |
| Semantic Scholar API | 开放 API | 适合检索论文、作者、引用关系和摘要信息。 | [Semantic Scholar API](https://www.semanticscholar.org/product/api) |
| OpenReview | 论文与评审平台 | AI 顶会最新论文、review、rebuttal 观察的重要入口。 | [OpenReview](https://openreview.net/) |
| arXiv API | 预印本 API | 获取预印本元数据与基础检索能力。 | [arXiv API](https://info.arxiv.org/help/api/) |
| Crossref REST API | DOI / 元数据 | 做 DOI、期刊论文元数据补齐时很有帮助。 | [Crossref REST API](https://www.crossref.org/documentation/retrieve-metadata/rest-api/) |
| Papers with Code | 平台 | 把论文和代码、benchmark、SOTA 页面关联起来。 | [Papers with Code](https://paperswithcode.com/) |

## 2. 研究向检索与阅读工作流

| 资源 | 场景 | 简述 | 链接 |
| --- | --- | --- | --- |
| RTI | 工作流 | 面向“找方向 -> 找论文 -> 做分析”的文献研究工作流。 | [yrc-better/RTI](https://github.com/yrc-better/RTI) |
| arxiv-translator | 中文翻译 | 从 arXiv LaTeX 源码生成中文版本，对国内科研同学尤其友好。 | [Leey21/arxiv-translator](https://github.com/Leey21/arxiv-translator) |
| paper-glance-skill | 深读加工 | 对单篇论文生成分析、导图、审稿意见。 | [CatVinci-Studio/paper-glance-skill](https://github.com/CatVinci-Studio/paper-glance-skill) |
| PaperQA2 | 文献问答 | 面向 scientific literature 的高精度 RAG。 | [Future-House/paper-qa](https://github.com/Future-House/paper-qa) |

## 3. PDF / 文档解析基础设施

| 资源 | 场景 | 简述 | 链接 |
| --- | --- | --- | --- |
| GROBID | 学术 PDF 结构化解析 | 把论文 PDF 转成结构化 TEI/XML，是做 citation / section 解析的经典底座。 | [kermitt2/grobid](https://github.com/kermitt2/grobid) |
| Marker | PDF 转 Markdown | 对把论文和技术文档转成 LLM 友好文本很有帮助。 | [VikParuchuri/marker](https://github.com/VikParuchuri/marker) |
| Docling | 文档解析 | 适合做 PDF、Office 文档、扫描文档的统一提取。 | [docling-project/docling](https://github.com/docling-project/docling) |

## 4. 给 ResearchOS 的组合建议

### 如果你要做 related work

- `paper-search-mcp`
- `OpenReview MCP`
- `OpenAlex MCP`
- `OpenAlex / Semantic Scholar API`

### 如果你要做单篇精读

- `arxiv-translator`
- `paper-glance-skill`
- `PaperQA2`
- `GROBID / Marker / Docling`

### 如果你要做 novelty check

- `OpenReview`
- `OpenAlex`
- `Crossref`
- `Papers with Code`

## 5. 使用提醒

- 用网页搜索补文献可以，但不要让网页搜索替代学术数据库。
- 论文摘要足够做粗筛，但真正做 claim 判断时最好读 PDF 或 LaTeX 源。
- 对关键结论，尽量保留 DOI、arXiv ID、OpenReview URL 或 OpenAlex ID。
